# 📚 Тренажер іспиту

Однофайловий веб-застосунок для підготовки до іспиту. Відкривається в будь-якому сучасному браузері — жодного сервера чи встановлення не потрібно.

## Можливості

- Випадкове питання з банку, зважене за частотою правильних відповідей (питання, з якими ви ще погано знайомі, з’являються частіше).
- **Не повторює те саме питання двічі поспіль** (якщо є альтернативи).
- Після правильної відповіді — лічильник серії `x/N`. Після помилки — показує правильну відповідь і скидає серію.
- Коли питання відповіли правильно **N разів поспіль**, воно вважається **опанованим** і прибирається з ротації.
- Клік по будь-якому варіанту після відповіді = миттєвий перехід до наступного питання (подвійний клік теж працює).
- Прогрес **автоматично зберігається після кожної відповіді** в `localStorage` браузера.
- Необов’язково: зберігання прогресу в **JSON-файл на диску** (кнопка `📁 Файл прогресу`, працює в Chrome/Edge).

## Структура проєкту

```
qi-test/
├── index.html              ← застосунок (відкривайте цей файл)
├── config.js               ← налаштування (поріг опанування тощо)
├── questions.js            ← банк питань (генерується зі questions_raw.txt)
├── questions_raw.txt       ← початковий текст питань
├── rebuild_questions.py    ← скрипт регенерації questions.js з questions_raw.txt
└── 1. Лоція/               ← окрема тема з власним набором питань
    ├── index.html
    ├── config.js
    └── questions.js
```

## Як користуватися

1. Відкрийте `index.html` подвійним кліком у браузері.
2. Відповідайте на питання. Прогрес зберігається автоматично.
3. (Опційно) Натисніть **📁 Файл прогресу**, щоб зберігати прогрес у JSON-файл на диску — зручно для перенесення між комп’ютерами.

## Налаштування (`config.js`)

```js
const CONFIG = {
    requiredCorrect: 10,        // скільки разів поспіль треба відповісти правильно
    resetStreakOnWrong: true,   // скидати серію при помилці
    avoidImmediateRepeat: true, // не показувати те саме питання двічі поспіль
    storageKey: "exam-trainer-progress-v1"
};
```

## Формат питань (`questions.js`)

```js
{
  id: 1,
  q: "Текст питання?",
  options: [
    "!Правильна відповідь",   // префікс «!» позначає правильну відповідь
    "Неправильний варіант",
    "Ще один варіант"
  ]
}
```

- `id` — унікальний номер. Прогрес прив’язаний до цього ID, тож не змінюйте його після початку навчання.
- Правильний варіант позначається **префіксом `!`** у тексті. «!» автоматично прибирається з відображення.
- Кількість варіантів — будь-яка (зазвичай 3–4).

## Регенерація `questions.js`

Якщо ви змінили `questions_raw.txt` (наприклад, додали нові питання), перебудуйте `questions.js`:

```powershell
python rebuild_questions.py
```

Скрипт:
1. Парсить усі питання з `questions_raw.txt`.
2. Записує `questions.js` із коректною структурою.
3. **Правильні відповіді треба позначити вручну**, додавши `!` перед текстом правильного варіанта.

> ⚠️ Регенерація повністю перезаписує `questions.js`. Усі вручну поставлені `!` буде втрачено — робіть резервну копію або вносьте зміни спочатку в `questions_raw.txt`.

## Формат `questions_raw.txt`

```
1. Текст першого питання?
- Варіант А;
- Варіант Б;
- Варіант В.
2. Текст другого питання?
- Варіант А;
- Варіант Б.
```

- Кожне питання починається з `N.` на початку рядка.
- Кожен варіант починається з `- `.
- Питання і варіанти можуть займати кілька рядків — скрипт автоматично їх об’єднує.
- Фінальні `;` та `.` у варіантах автоматично прибираються.

## Скидання прогресу

- **DevTools (F12)** → вкладка **Application** → **Local Storage** → видалити ключ `exam-trainer-progress-v1`.
- Або в консолі браузера:
  ```js
  localStorage.removeItem("exam-trainer-progress-v1");
  ```

## Де зберігається прогрес

- **За замовчуванням:** у `localStorage` поточного браузера. Прив’язаний до браузера й шляху файлу — інший браузер чи приватний режим = інший прогрес.
- **Режим «файл на диску»:** натисніть `📁 Файл прогресу` — прогрес буде автоматично записуватися в обраний JSON-файл. Працює в Chrome/Edge (Firefox/Safari — тільки `localStorage`).
